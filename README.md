## Что это

Готовая система, которая каждые 30 минут ищет новые сделки на рынке, отделяет их
от старых новостей и собирает **ежедневный banker brief**.

Вместо десятка новостных вкладок ты получаешь один экран с ответами на три вопроса:

1. **Что произошло?** — новая M&A, ECM или DCM-сделка.
2. **Почему это важно?** — какой banker angle затрагивает событие.
3. **Что делать дальше?** — какой deliverable нужно обновить аналитику.

Работает на **GitHub Actions**, официальных раскрытиях **MOEX**, RSS **Банка России**,
IR-сайтах эмитентов и Google News как резервном discovery-слое. Не нужен сервер,
Docker или платный API.

## Для кого и зачем

- 🎓 **Стажёр или junior analyst в investment banking** — быстрее готовить morning update и не пропускать новые сделки.
- 🏦 **Небольшая M&A / advisory-команда** — следить за рынком без Bloomberg и дорогих data terminals.
- 🧭 **Corporate development** — замечать приобретения, размещения и новые сценарии по компаниям покрытия.
- 📚 **Кандидат в finance** — показать на собеседовании работающий data pipeline, deal thinking и автоматизацию.

Проект закрывает рутинную задачу: **собрать свежий deal flow, убрать информационный шум и превратить новость в конкретное действие аналитика**.

## Возможности

**Мониторинг сделок:**

- 🤝 **M&A** — приобретения, продажи активов, слияния и закрытие сделок
- 📈 **ECM** — IPO, SPO и размещения акций
- 💳 **DCM** — выпуски облигаций и долговое финансирование
- 🌐 **Market-wide radar** — поиск по рынку, а не только по трём выбранным компаниям
- 🕒 **Только свежие данные** — 24 часа в будни, 72 часа для weekend/Monday catch-up

**Banker workflow:**

- 🗓 два независимых режима: свежая лента за 24/72 часа и постоянный архив сделок
- 🔟 на главном экране всегда показываются 10 последних ключевых M&A / ECM / DCM без повторов одной сделки
- 🆕 новые события отделяются от уже просмотренных
- 🎯 сигнал связывается с активной deal hypothesis
- ✅ система формирует очередь конкретных analyst actions
- 🔗 каждая карточка содержит дату, источник и ссылку на материал
- 📊 MOEX market tape показывает цену, изменение и оборот компаний покрытия
- ☁️ dashboard автоматически обновляется и публикуется через GitHub Pages
- 🏛 официальные MOEX / ЦБ / issuer IR записи получают приоритет над новостными агрегаторами

**Deal execution toolkit:**

- 🧾 **Deal card** — стороны, статус, сумма, доля, форма оплаты, advisors и rationale / use of proceeds
- 🗂 **Precedent transactions database** — накопительная JSON/CSV-база без дублей, с датой и источником
- 📐 **Precedent analytics** — EV/Revenue, EV/EBITDA и медианы только по валидным M&A-наблюдениям
- 📥 **Excel-export** — banker-style workbook с формулами, Dashboard, Deal Card, полной базой и Sources & QA
- ✅ неизвестные параметры остаются `Not disclosed`: система не придумывает отсутствующую экономику сделки

## Что ты получишь

Пример карточки в утренней сводке:

```text
P1 · DCM · SELECTEL

Событие: компания разместит облигации на 5 млрд ₽
Почему важно: меняется debt profile и набор долговых аналогов
Действие: обновить debt comps и проверить ближайшие погашения
Источник: публичная новость со ссылкой и датой
```

Если свежих сделок нет, live-лента честно показывает пустой день. При этом блок
**«10 последних ключевых сделок»** не исчезает: он берётся из постоянного 90-дневного
архива, который ежедневно пополняется и дедуплицируется по транзакции.

## Быстрый старт

1. Нажми **Use this template** или клонируй репозиторий.
2. Измени компании, запросы и deal hypotheses в [`config.json`](config.json).
3. Запусти live-сборку:

```bash
python3 run.py --live
```

4. Открой `output/deal_markets_brief.html`.

В dashboard доступны выгрузки:

- `output/precedent_transactions.csv` обновляется при каждом запуске;
- `output/precedent_transactions.xlsx` — готовый Excel-файл для анализа и precedent screen.

HTML, JSON и CSV обновляются в GitHub Actions в течение рабочего дня. `.xlsx`
пересобирается отдельной ежедневной автоматизацией в 08:45 по Москве, проходит
визуальный QA и публикуется в тот же GitHub Pages release. Скрипт сборки оставлен
в репозитории; для локальной пересборки нужен совместимый Codex workspace с
`@oai/artifact-tool`.

Ничего устанавливать не требуется: достаточно Python 3.11+. Полная инструкция —
[`docs/SETUP.md`](docs/SETUP.md).

## Автономная работа 24/7

Workflow [`.github/workflows/deal-desk.yml`](.github/workflows/deal-desk.yml):

- запускает тесты;
- получает свежий deal flow;
- сохраняет baseline для change detection;
- публикует dashboard в GitHub Pages;
- повторяет цикл по будням с 08:30 до 18:30 по Москве.

Дополнительно локальная Codex-автоматизация `Deal Markets Copilot — daily XLSX`
каждый день в 08:45 обновляет `.xlsx`, запускает тесты и публикует проверенный файл.

Для облачного запуска включи **Settings → Pages → Source: GitHub Actions** и запусти
workflow вручную один раз.

## Безопасность данных

- 🔓 Репозиторий публичный: в нём находятся только код и данные из открытых источников.
- 🔐 Токены, email и локальные пути пользователя не хранятся в файлах проекта.
- 🚫 `.env`, приватные ключи и сертификаты исключены через `.gitignore`.
- 🛡 GitHub Actions получает только минимально необходимые разрешения.
- 🔗 В dashboard разрешены только внешние ссылки `http/https`.
- 📭 Telegram по умолчанию выключен; секреты следует хранить только в GitHub Secrets.
- 💼 Инструмент не подключается к брокерскому счёту и не совершает сделки.

## Ограничения

- Google News RSS может публиковать событие с задержкой или вести через агрегатор;
  такие записи маркируются `unverified` и уступают официальному первичному источнику.
- Статус `unverified` означает, что аналитик ещё не проверил первичный документ.
- Небольшой coverage universe не гарантирует новую сделку каждый день.
- Мультипликатор считается только при явно раскрытых EV и LTM financials в одной валюте;
  иначе отображается `N/M`.
- Это screen-grade monitoring tool и precedent screen, а не fairness opinion.
